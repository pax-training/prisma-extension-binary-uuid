# Support

## Questions + discussion

- [GitHub Discussions](https://github.com/pax-training/prisma-extension-binary-uuid/discussions) — design questions, usage help, ideas
- [Prisma Discord](https://pris.ly/discord) — the broader Prisma community is active here; tag @mav2287 if you want a maintainer response

## Bug reports

File on the GitHub issue tracker using the [Bug report](https://github.com/pax-training/prisma-extension-binary-uuid/issues/new?template=bug_report.yml) template.

## Response-time SLA

This is a maintained open-source project, not a paid product. Best-effort
SLAs for a single maintainer:

| Severity | Description                       | Target initial response |
| -------- | --------------------------------- | ----------------------- |
| Critical | Data corruption, data leak        | 48 hours                |
| High     | Query failures, incorrect results | 1 week                  |
| Medium   | Docs gaps, ergonomic issues       | Best effort             |
| Low      | Nice-to-haves                     | Best effort             |

## Contract support

If your organization needs faster response times or custom features, email
james@paxtraining.com to discuss.

## What's NOT supported

- Real-time debugging over chat
- Private repositories' code
- Integration with proprietary Prisma forks
- Back-porting fixes to unsupported major versions
